# yoloAI docs

Docs are organised by **role** — the layer you are operating in. Pick your tier.

| Tier | Start here |
| --- | --- |
| **Users** — running yoloAI | [`GUIDE.md`](GUIDE.md) |
| **Integrators** — embedding yoloAI as a library or daemon | [`integrators/`](integrators/) |
| **Contributors** — changing yoloAI itself | [`contributors/`](contributors/) |

## This tier (users)

- **[`GUIDE.md`](GUIDE.md)** — the full usage reference: commands, flags, workdir modes,
  agents and models, configuration, sandbox state, security.
- **[`BREAKING-CHANGES.md`](BREAKING-CHANGES.md)** — every breaking change during beta, with
  rationale and migration steps. Entries land under `## Unreleased` and are stamped to a
  version at release.
- **[`ROADMAP.md`](ROADMAP.md)** — what's planned: agents, network isolation, profiles.

yoloAI is in **public beta**. Breaking changes are allowed and are always recorded in
`BREAKING-CHANGES.md`.
