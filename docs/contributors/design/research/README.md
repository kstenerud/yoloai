| [Crash-safe migration](crash-safe-migration.md) | **Verified 2026-06-30 (backs DF68 / [plans/crash-safe-migration.md](../plans/crash-safe-migration.md)).** Prior art for crash-safe `system migrate`: SQLite (rollback-journal/WAL, the delete-is-commit point, hot-journal detection), POSIX atomic-rename + fsync idiom (write-temp-same-dir → fsync → rename → fsync-dir; macOS `F_FULLFSYNC`), ARIES (WAL ordering + redo-idempotency + the CLR insight, with DPT/checkpoints/locking flagged as overkill), Nix/OSTree (build-alongside + atomic pointer-swap; retained generation *is* the snapshot; cheap CoW/hardlink retention), dpkg/rpm (**the cautionary tale** — both bet on roll-forward/resume; rpm shipped reverse-replay rollback and deleted it as "too unreliable"). Unanimous signal: **build-alongside + pointer-swap > mutate-in-place + restore-backup; forward=resume-first, rollback=wholesale snapshot-restore-only; journal=coarse per-sandbox state, not a fine WAL.** Maps to all six draft open questions. |

| [Agent proxy support](agent-proxy-support.md) | **Verified mid-2026 (resolves Q37).** Whether each shipped agent honors `HTTP_PROXY`/`HTTPS_PROXY` (Claude/OpenCode yes; Gemini/Codex partial — Codex disables it *in its own sandbox*; Aider only with `DISABLE_AIOHTTP_TRANSPORT`). The design conclusion: env-proxy is **never** a containment boundary for an untrusted agent, and transparent intercept *alone* is bypassable (`NET_ADMIN`/UDP/IPv6) — the hostile-containment primitive is a **default-deny egress netns + forced proxy on a different principal/namespace**. Backs [netpolicy.md](../netpolicy.md) §Hostile. |
| [Agent self-update](agent-self-update.md) | **Verified 2026-06-26.** Whether the substrate must carry `AgentInstallMethod`. Claude's `installMethod`'s *only* load-bearing use is the auto-updater (docs + [#28625](https://github.com/anthropics/claude-code/issues/28625)); the PATH/`~/.local/bin/claude` warning we patch around is that machinery on a host-vs-image mismatch. `DISABLE_AUTOUPDATER=1` (settings.json `env`) makes `installMethod` **inert**. Decision: **delete `AgentInstallMethod`** from the substrate, disable Claude auto-update as static agent self-config, strip the stale `installMethod` from the seeded file — no manifest needed. Keep `hasCompletedOnboarding:true` seeded ([#4714](https://github.com/anthropics/claude-code/issues/4714)). Backs D97 / arch-principles §4. |

| [Egress-broker host reachability](egress-broker-host-reachability.md) | **Linux-verified 2026-06-28 (feeds D106 / egress-proxy 2b-2).** How an in-sandbox agent reaches the host-side key-injector, per backend, and where the injector binds so only the sandbox — not the LAN — reaches it. Three classes: bridge+NAT (docker/podman → bridge gateway IP), VM+CNI (containerd/apple → CNI gateway), host-process (seatbelt → `127.0.0.1`); tart unverified. `HostFromContainer` is **advisory-only** today (no `ExtraHosts` wired → `host.docker.internal` doesn't resolve on Linux Engine). Decision: **gateway-IP-for-both** via a backend `InjectorReach{BindHost,DialHost}` seam; the split is kept because **Docker Desktop** (LinuxKit VM) needs `host.docker.internal` dial + Mac-host bind. Isolation blocks the gateway unless allowlisted (deferred — Phase 1 is open-egress). Includes a runnable **Mac spike brief**. |
| [Secure credentials](secure-credentials.md) | **Verified 2026-06-25.** Best-practice secure credential delivery against the **duped-agent** (confused-deputy / lethal-trifecta) threat. Ecosystem survey (env-var-everywhere; no shipping agent brokers the key out; default-deny egress rare; isolation≠egress); verified injection→exfil incidents; prior-art patterns (short-TTL, broker-vends, ephemeral-mount-not-env, destination-scoped, unforgeable-mint à la IMDSv2); Anthropic WIF (`sk-ant-oat01`, 1h, verified); egress-proxy/token brokers; ssh-agent forwarding. Conclusion: the **egress-proxy/broker = netpolicy's reserved strategy** (one boundary point for creds+network) is the highest-impact *novel* control; layer with a hardened narrow allowlist, egress logging + canaries, and the copy/diff/apply review gate for the in-session residual. Ownership split: yoloai owns proxy/broker/egress + a short-lived-cred caller hook; the embedder mints/scopes/revokes. Feeds **DF38** + [netpolicy.md](../netpolicy.md). |

| [Shared-state concurrency](shared-state-concurrency.md) | **Verified 2026-06-15.** Daemon vs daemonless (Docker single-arbiter SPOF vs Podman's daemonless locks+SQLite — works but not free: lock-exhaustion, `database is locked`, the complexity moves not vanishes) + file-locking wisdom (`flock` not `fcntl`; atomic-rename for lock-free reads; the no-multi-file-atomic-commit limit; SQLite's NFS hard-limit; the envelope where file locks suffice). Verdict: yoloAI's file-locked multi-tool model is sound *inside the envelope* (single host, low contention, single-doc-per-domain), daemon stays optional. Backs [persistence-helper.md](../persistence-helper.md) / [D87](../../decisions/working-notes.md). |
| [Container init / keep-alive delineation](container-init-delineation.md) | **Verified 2026-06-14.** Where a substrate stops and the OS begins — the Tier 0–3 spectrum (isolation → PID-1 hygiene → supervision → orchestration), the PID-1 zombie/signal problem + thin-init fix (tini/dumb-init/`--init`), Tier-2 supervisors we must NOT build, and the Kubernetes gotcha museum (the `pause` neutral-init, init vs sidecar — KEP-753's 4-year saga, probes, shareProcessNamespace, restartPolicy/backoff). Backend audit: VM backends already keep the env up via the guest OS init; Seatbelt-P1 already uses `tail -f /dev/null`; only Docker/Podman weld the agent session to PID 1 (DF31). Recommends: substrate contract = "up + exec/transfer", keep-alive backend-native (`KeepAliveModel` property), agent launched via exec, monitor as a probe. Feeds [public-layering](../plans/public-layering.md), Q103, DF31. |
| [Tart `-xcode` base image](tart-xcode-base-image.md) | **A/B verified 2026-06-09; feature uncommitted, adoption undecided.** Cirrus `-xcode` images bake Xcode + default iOS runtime → "image mode" skips the host-Xcode mount + in-VM runtime download.
