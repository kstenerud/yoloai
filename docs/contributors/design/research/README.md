| [Agent proxy support](agent-proxy-support.md) | **Verified mid-2026 (resolves Q37).** Whether each shipped agent honors `HTTP_PROXY`/`HTTPS_PROXY` (Claude/OpenCode yes; Gemini/Codex partial — Codex disables it *in its own sandbox*; Aider only with `DISABLE_AIOHTTP_TRANSPORT`). The design conclusion: env-proxy is **never** a containment boundary for an untrusted agent, and transparent intercept *alone* is bypassable (`NET_ADMIN`/UDP/IPv6) — the hostile-containment primitive is a **default-deny egress netns + forced proxy on a different principal/namespace**. Verified: live `:rw`/`:ro` virtiofs, in-guest overlayfs (CAP_SYS_ADMIN), in-guest iptables allowlist, clean `--format json`. Full env-var catalog; daemon via XPC (no socket env var). Conclusion: Docker-class caps + stronger isolation, reuses profile images. |

| [Base image candidates](base-image-candidates.md) | **Parked 2026-05-24.** Survey of pre-built bases (devcontainers/universal, gitpod/workspace-full, buildpack-deps, …) as replacements for `debian:bookworm-slim` under `yoloai-base`. Gotchas: Docker-in-Docker, `yoloai` uid 1001, AI-agent layer doesn't compress. Tart/Seatbelt not applicable. Recommendation: defer; only honest win is `buildpack-deps:bookworm` and it's not worth the churn. |
| [Dind storage drivers](dind-storage-drivers.md) | Nested `dockerd` storage driver across Linux / Docker Desktop / OrbStack / Podman Machine. **Linux verified 2026-06-05**: overlay2 nests on a real-fs `/var/lib/docker` (O3) — fast + low-disk; fuse-overlayfs pin only papers over the overlay-on-overlay backing. Recommends overlay2 + real-fs volume, vfs fallback. macOS columns + O1 EINVAL root-cause **pending a Mac host** — includes a full repro recipe for a Mac agent. |
| [Shared-state concurrency](shared-state-concurrency.md) | **Verified 2026-06-15.** Daemon vs daemonless (Docker single-arbiter SPOF vs Podman's daemonless locks+SQLite — works but not free: lock-exhaustion, `database is locked`, the complexity moves not vanishes) + file-locking wisdom (`flock` not `fcntl`; atomic-rename for lock-free reads; the no-multi-file-atomic-commit limit; SQLite's NFS hard-limit; the envelope where file locks suffice). Verdict: yoloAI's file-locked multi-tool model is sound *inside the envelope* (single host, low contention, single-doc-per-domain), daemon stays optional. Backs [persistence-helper.md](../persistence-helper.md) / [D87](../../decisions/working-notes.md). |
| [Container init / keep-alive delineation](container-init-delineation.md) | **Verified 2026-06-14.** Where a substrate stops and the OS begins — the Tier 0–3 spectrum (isolation → PID-1 hygiene → supervision → orchestration), the PID-1 zombie/signal problem + thin-init fix (tini/dumb-init/`--init`), Tier-2 supervisors we must NOT build, and the Kubernetes gotcha museum (the `pause` neutral-init, init vs sidecar — KEP-753's 4-year saga, probes, shareProcessNamespace, restartPolicy/backoff). Backend audit: VM backends already keep the env up via the guest OS init; Seatbelt-P1 already uses `tail -f /dev/null`; only Docker/Podman weld the agent session to PID 1 (DF31). Recommends: substrate contract = "up + exec/transfer", keep-alive backend-native (`KeepAliveModel` property), agent launched via exec, monitor as a probe. Feeds [public-layering](../plans/public-layering.md), Q103, DF31. |
| [Tart `-xcode` base image](tart-xcode-base-image.md) | **A/B verified 2026-06-09; feature uncommitted, adoption undecided.** Cirrus `-xcode` images bake Xcode + default iOS runtime → "image mode" skips the host-Xcode mount + in-VM runtime download. Works end-to-end (iOS tests pass, no host mount), but base is ~87 GB vs ~32 GB. Documents the size tradeoff, the arm64 swift-syntax pin, and bugs found (promote-delete-running, sandbox-setup None-deref, parseVMList column drift, cache-key collision; one unresolved base-disappearance). |
